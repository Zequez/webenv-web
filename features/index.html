Features

Here are the actual functionalities that are currently working.

- Mainframe
  - Electron-based UI for self-management of the Field
    - Sub-Repo management
      - Add new empty local sub-repos
      - Clone to local sub-repos from a Git URL
      - Bi-dimensional columns board for organizing repos
      - Individual Sub-Repos
        - Start/Stop dev server with port auto-assign
        -
    - Field-Repo management
      - See field origin URL
      - Check for updates, sync code
      - See local changes; commit local changes
      - See list of latest commits

  - Field Management Local Servers
    - GIT server
    - SubRepo spinning server
      Uses the Meta Vite Driver
      Allows you to request turning on and off certain sub-repos dev servers
      and it returns the localhost address with it's custom port
      It also provides a websocket that notifies with a list of active servers
      Saves the repos ports state and on/off state on a persisted file
      and it re-starts the repos that were open when the server was last closed
    - Tunneling server
      Allows you to declare arbitrary files with
      functions as individually callable from a
      web client and get their return data or create
      a websocket to continuosly stream them.
      Individual repos may create tunnel files for their own purposes in order to
      access the OS execution environment from the browser.
      [Will also auto-generate typings for client-side code]
    - Files server: Used on some places but soon to be deprecated by a Tunnels Library
  - SubRepo Drivers System
    - Meta Vite Driver
      - UnoCSS with Tailwind, Icons, Fonts presets, and some in-house utilities
      - Svelte
      - Auto-Import plugin
      - Shortcut path aliases for Local Field Libraries
      - Iconify
      - ImageTools
      - YAML
      - Svelte pre-rendering for building
      - CNAME generator from wenv.yaml configuration file
    - Publishing build to separate www branch for Github Pages

- Curated Library
  - Local Field Library
    - Lives on @/center
    - Common utility libraries
      - Copy to clipboard
      - Monitor Pinch Zoom for mobile
      - Observers Shortcuts
        - Observe Children Added-Removed
        - Resize Observer
      - Maximize Font Size
      - onTripleKey
      - Components
        - Light/Dark Toggle
        - Internationalization
          - enes store and component
      - Svelte
        - Directives
          - Set CSS Variables
        - Stores
          - LocalStorage-persisted State
          - Draggable Handler
    - Substrates

      Substrates are interactive components
      that facilitate a specific for a creative
      experience. Substrates can take up
      whole pages, or just specific parts.

      - Linear Web
      - Toroidal Web
      - Navigatable Canvas Web
      - Empty
      - Toroidal Web
      - ...«canvas» Sandbox
      - ...TLDraw

    - Visual Collections
      - Generative

        Parameterized canvas-based
        pattern generators

        - Starry Sky
        - Grass
        - Clouds
        - UnderGrass

      - Patterns and Textures
      - SVG Separators
