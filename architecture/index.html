Architecture

Here I'm attempting to give an overview of the architectural decisions of the Web Environment that maket it different from traditional web repositories.

  • There is no production code
  • All sub-repositories run on the same environment
  • Substrates as a new distinction
  • Mainframe: Web Environment self-management and sub-repos management
  • Meta Vite Configurator
  • Automatic port assignment
  • Web Projecting
  • Custom CLI rather than package.json script
  • Shared curated package.json and config for every project
